  <action>
    **Note:** The stock-adjustments schema file was created by Plan 01. This task uses the existing schema -- it does NOT create or modify stock-adjustments.ts.

    **Extend stock tRPC router:**
    - Add `stock.adjust` mutation (warehouseProcedure): Input validated with Zod -- batchId (uuid), productId (uuid), quantityChange (integer, non-zero), reason (enum: damage, returns, cycle_count_variance, other), notes (string optional). In a transaction: (1) insert stockAdjustments record, (2) update batch quantityAvailable by quantityChange, (3) update stockItems quantity, (4) if quantityAvailable drops to 0, set batch status to 'depleted'. Throw if adjustment would make quantityAvailable negative.
    - Add `stock.adjustmentHistory` query (warehouseProcedure): Returns recent adjustments (last 100) joined with batch (torkeBatchId) and product (name, sku). Ordered by createdAt DESC.

    **src/components/wms/StockAdjustmentForm.tsx:** "use client" component.
    - Form using react-hook-form + Zod validation
    - Fields: Product selector (searchable dropdown), Batch selector (filtered by selected product, shows torkeBatchId + available qty), Quantity Change (number input -- negative for reductions), Reason (select: Damage, Returns, Cycle Count Variance, Other), Notes (textarea, optional)
    - Submit calls stock.adjust mutation
    - Success: show sonner toast "Stock adjusted: {product} batch {batchId} by {qty}"
    - Error: show sonner toast with error message (e.g. "Cannot reduce below zero")
    - After successful adjustment, invalidate stock.dashboard query to refresh data

    **src/app/(wms)/stock/adjustments/page.tsx:**
    - Title "Stock Adjustments"
    - Split layout: adjustment form on left (or top on mobile), recent adjustment history on right (or bottom)
    - History table columns: Date, Product, Batch, Change (+/-), Reason, Notes, Adjusted By
    - Positive changes shown in green, negative in red
    - Link back to stock dashboard

    Per user decision: stock adjustments are immediate with no approval workflow. All logged for audit trail.
