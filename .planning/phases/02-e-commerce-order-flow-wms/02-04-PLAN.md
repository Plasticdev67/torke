    **Extend orders tRPC router** (warehouseProcedure):
    - `orders.queue`: Returns orders in statuses [confirmed, allocated, picking, packed] with customer name, order number, date, line count, total. Ordered by confirmedAt ASC (oldest first). Filterable by status.
    - `orders.getPickList`: Input: orderId. Returns order with lines, each line with product (name, sku), quantity, and allocations (torkeBatchId, batchQuantity). Allocations come from orderLineAllocations joined to batches. This proves FIFO enforcement (WMS-02) -- pick list shows the batch(es) that allocateFIFO selected (oldest first).
    - `orders.startPicking`: Input: orderId. Transitions allocated -> picking. Sets pickedAt timestamp on allocations.
    - `orders.completePacking`: Input: orderId. Transitions picking -> packed.

    - Polling or refetch on focus to show new orders

In transaction: (1) transition packed -> dispatched, (2) update order with dispatchType, trackingNumber/consignmentNumber, dispatchedAt, (3) update orderLineAllocations.dispatchedAt for all allocations, (4) trigger cert pack generation asynchronously. For now, the cert pack trigger is a placeholder call to a certpack-service function (implemented in Plan 05).
