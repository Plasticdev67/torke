<task type="auto">
  <name>Task 2: Orders tRPC router with server-side price validation</name>
  <files>
    src/server/trpc/routers/orders.ts,
    src/server/trpc/router.ts
  </files>
  <action>
    **src/server/trpc/routers/orders.ts:** (protectedProcedure unless noted)
    - `orders.create`: Input: deliveryAddressId, paymentMethod (card|credit|bacs), poNumber?, items (array of {productId, quantity}). In transaction: (1) fetch product prices from products.pricePence column (CRITICAL: never trust client prices), (2) validate all products have pricePence set (throw if any product has null price), (3) calculate subtotal (sum of qty * pricePence), VAT (20% of subtotal), total, (4) call createOrder from order-service with server-validated prices, (5) return orderId and orderNumber.
    - `orders.getById`: Input: orderId. Returns order with lines, address, allocations. Only allows viewing own orders (or warehouse role).
    - `orders.confirmPayment`: (warehouseProcedure) Input: orderId. Admin confirms BACS payment received. Transitions awaiting_payment -> confirmed, then runs allocateOrderStock.
    - `orders.getCreditAccount`: Returns current user's credit account status (or null if none).

    **Update src/server/trpc/router.ts:** Wire orders router (if not already wired in Task 1).

    **Server-side price validation detail:** The orders.create mutation receives only productId and quantity from the client. It fetches pricePence from the products table for each item. This prevents price manipulation. If a product's pricePence is null, throw an error: "Product {name} does not have a price set. Please contact Torke."
  </action>
