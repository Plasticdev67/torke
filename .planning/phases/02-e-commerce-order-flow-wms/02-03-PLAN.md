    **src/server/trpc/routers/addresses.ts:** (protectedProcedure for all)
    - `addresses.list`: Returns all addresses for current user, ordered by isDefault DESC then createdAt DESC
    - `addresses.create`: Input: name, addressLine1, addressLine2?, city, county?, postcode, country?, siteContactName?, siteContactPhone?. Sets isDefault=true if first address. Validates postcode format loosely (UK format).
    - `addresses.update`: Input: id + all fields. Only allows updating own addresses.
    - `addresses.delete`: Input: id. Only allows deleting own addresses.
    - `addresses.setDefault`: Input: id. Unsets all other defaults for user, sets this one.

    **src/server/trpc/routers/orders.ts:** (protectedProcedure)
    - `orders.create`: Input: deliveryAddressId, paymentMethod (card|credit|bacs), poNumber?, items (array of {productId, quantity}). In transaction: (1) fetch product prices from DB (never trust client), (2) calculate subtotal/VAT(20%)/total, (3) call createOrder from order-service, (4) return orderId and orderNumber.
    - `orders.getById`: Input: orderId. Returns order with lines, address, allocations. Only allows viewing own orders.
    - `orders.confirmPayment`: (warehouseProcedure) Input: orderId. Admin confirms BACS payment received. Transitions awaiting_payment -> confirmed, then runs allocateOrderStock.
    - `orders.getCreditAccount`: Returns current user's credit account status (or null if none).

    **src/server/services/payment-service.ts:**
    - `createStripeCheckoutSession(orderId, orderRef, lineItems, customerEmail)`: Creates Stripe Checkout Session per research code example. mode: 'payment', currency: 'gbp', unit_amount in pence. metadata includes orderId and orderRef. success_url: /checkout/success?order={orderRef}, cancel_url: /basket. Returns session URL for redirect.
    - `validateCreditAccount(userId, totalPence)`: Checks user has approved credit account with sufficient available credit (creditLimitPence - creditUsedPence >= totalPence). Returns { valid, account, availableCredit } or throws.
    - `processPayment(tx, orderId, method, userId)`: Strategy pattern per research. Card: create Stripe session, transition to awaiting_payment, return sessionUrl. Credit: validate credit, deduct from available credit, transition to confirmed. BACS: transition to awaiting_payment (proforma generated separately).
    - Install stripe: `npm install stripe @stripe/stripe-js`
    - Use env: STRIPE_SECRET_KEY, STRIPE_WEBHOOK_SECRET

    **src/app/api/stripe/webhook/route.ts:**
    - POST handler per research code example
    - Verify signature with stripe.webhooks.constructEvent(body, sig, STRIPE_WEBHOOK_SECRET)
    - Handle 'checkout.session.completed' event: extract orderId from metadata, transition order awaiting_payment -> confirmed, run allocateOrderStock, store stripePaymentIntentId on order
    - Idempotency: check order status before transition -- if already confirmed, skip (Stripe can send duplicate webhooks)
    - Return 200 on success, 400 on invalid signature
    - CRITICAL: Use req.text() for raw body (not req.json()). App Router does not need bodyParser config.
