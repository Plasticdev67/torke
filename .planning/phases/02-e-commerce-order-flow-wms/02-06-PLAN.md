Output: Email service with Resend, 2 email templates, email wiring into order lifecycle, account dashboard, order history/detail pages, reorder functionality.

    - Props: orderRef, customerName, dispatchType, trackingNumber?, consignmentNumber?, carrierName?, certPackUrl?, lines (product, qty, batchIds[])
    - Structure:
      - Header: "Your Order Has Been Dispatched"

      - Footer: Torke contact details

    **src/server/services/email-service.ts:**
    - Initialize Resend client with RESEND_API_KEY
    - `sendOrderConfirmation(orderId: string)`: Query order details, render OrderConfirmationEmail, send via Resend. From: "Torke <orders@torke.co.uk>" (or use env RESEND_FROM_EMAIL). Handles both confirmed orders and BACS awaiting payment orders.
    - `sendDispatchNotification(orderId: string)`: Query order details including cert pack. If certPackKey exists, fetch cert pack PDF from R2. If file size < 10MB, attach to email. If >= 10MB, generate a presigned download URL (valid 7 days) and include in email. Render DispatchNotificationEmail, send via Resend.
    - Error handling: log failures but don't throw (email failure should not block order flow). Use try/catch with console.error.

    **Wire email calls into src/server/trpc/routers/orders.ts:**
    - In the `orders.create` mutation: after successful order creation, call `sendOrderConfirmation(orderId)` in a fire-and-forget pattern (do not await -- use `.catch(console.error)` to prevent blocking). This sends confirmation for all payment methods.
    - In the `orders.dispatch` mutation (added by Plan 04): after successful dispatch transition, call `sendDispatchNotification(orderId)` in a fire-and-forget pattern. This sends the dispatch email with tracking info and cert pack (TRACE-10).
    - Import sendOrderConfirmation and sendDispatchNotification from email-service.ts at the top of orders.ts.
    - IMPORTANT: Email calls must not block or fail the order/dispatch mutation. If email fails, the order should still succeed.

    **src/__tests__/email/email-service.test.ts:** Fill in test stubs from Wave 0.
    - Mock Resend client
    - Test sendOrderConfirmation sends email with correct order details
    - Test sendDispatchNotification includes tracking info
    - Test sendDispatchNotification attaches cert pack when under 10MB
    - Test email failure does not throw (catches and logs)

    - `orders.myOrders`: Returns current user's orders with status, orderNumber, totalPence, createdAt, line count. Ordered by createdAt DESC. Paginated (limit/offset).
    - `orders.myOrderDetail`: Input: orderId. Returns full order with lines, allocations (batch IDs), delivery address, invoice (if exists), certPackKey. Validates user owns the order.
    - `orders.reorder`: Input: orderId. Returns the order's line items (productId, productName, sku, quantity, current price from products table) so they can be added to cart. Does NOT create a new order -- just returns the items for the client to add to cart.
    - `orders.accountSummary`: Returns spending summary for current user: total orders, total spent (all time), monthly spending (current month), top products (by order frequency), credit account status (if applicable), count of available cert packs.
