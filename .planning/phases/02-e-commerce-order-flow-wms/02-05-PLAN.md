<objective>
Build PDF generation services for cert packs, invoices, and BACS proforma invoices using pdf-lib. Cert packs merge a Torke-branded cover page with original supplier 3.1 cert PDFs. Invoices include batch references per line item as a Torke differentiator. Proforma invoices are generated for BACS orders with bank details.

Purpose: Cert packs are the traceability deliverable -- legally defensible documents containing original 3.1 certs. Invoices with batch references are a unique value proposition. BACS proforma invoices fulfill the user decision "BACS via proforma invoice with bank details."
Output: Cert pack service, invoice service (with proforma), download API endpoints for both.
</objective>

  <action>
    Install pdf-lib: `npm install pdf-lib` (if not already installed).

    **src/server/services/certpack-service.ts:**
    - `generateCertPack(orderId: string): Promise<Buffer>` -- main function
    - Steps:
      1. Query order with lines, allocations, batches, supplierBatches, millCerts, suppliers, products, and delivery address. Use Drizzle joins or query builder.
      2. **Create cover page** (A4 595x842 points):
         - Torke logo area (draw text "TORKE" in bold, or embed a simple logo placeholder -- actual logo embedding can use pdf-lib's embedPng if a PNG logo exists at public/torke-logo.png)
         - Title: "Certificate Pack" in large font
         - Order details section: Order Number, Order Date, Customer Name, Delivery Address, PO Number
         - Traceability table with columns per user decision: Line # | Product | Qty | Torke Batch ID | Supplier Batch | Manufacturer | Heat Number | Cert Page Ref
         - "Cert Page Ref" column shows the page number(s) where the cert appears in the merged document
         - Footer: "Generated by Torke - {date}. Original certificates appended unaltered."
         - Use StandardFonts (Helvetica, Helvetica-Bold) -- no custom font embedding needed
         - Draw table with lines using page.drawLine() and page.drawText()
      3. **Merge original cert PDFs:**
         - For each unique cert URL across allocations, fetch the PDF from R2 using GetObjectCommand
         - Load each cert with PDFDocument.load(certPdfBytes)
         - Copy all pages with mergedDoc.copyPages(certDoc, certDoc.getPageIndices())
         - Add each copied page to the merged document
         - Track page numbers for the "Cert Page Ref" column on the cover page
         - CRITICAL: Per user decision, original certs must be appended UNALTERED. Use copyPages only, never modify source PDFs.
      4. Build final merged document: cover page(s) first, then cert pages
      5. Save and return as Buffer
      6. Upload cert pack PDF to R2 with key `certpacks/{orderId}.pdf`
      7. Update order.certPackKey with the R2 key

    - Handle edge cases:
      - If cert PDF fetch from R2 fails (404), skip that cert and note "Cert unavailable" in the table
      - If cover page content exceeds one page, continue on a second page
      - Log cert pack generation time for monitoring

    **src/app/api/certpack/[orderId]/route.ts:**
    - GET handler (protected -- verify user owns order or is warehouse role)
    - Check if certPackKey exists on order -- if yes, stream from R2
    - If no certPackKey, generate on-demand by calling generateCertPack
    - Return with headers: Content-Type: application/pdf, Content-Disposition: attachment; filename="CertPack-{orderNumber}.pdf"

    **src/__tests__/certpack/certpack.test.ts:** Fill in test stubs from Wave 0.
    - Test generateCertPack produces a valid PDF buffer
    - Test cover page includes traceability table columns
    - Test missing cert PDFs are handled gracefully (not thrown)
    - Test cert pack is uploaded to R2 and order.certPackKey updated

    **generateInvoice(orderId: string): Promise<Buffer>** -- main invoice function
    - Steps:
      1. Query order with lines, products, allocations (for batch refs), delivery address, user profile (company name), credit account (if applicable for payment terms)
      2. Generate invoice number: `INV-YYYYMM-NNNNNN` (same sequence pattern as order numbers, separate counter)
      3. Create invoice record in invoices table with amounts and invoice number
      4. **Create invoice PDF** (A4):
         - Header: Torke company details (name, address, VAT number placeholder, email, phone)
         - "INVOICE" title
         - Invoice details: Invoice Number, Invoice Date, Due Date (immediate for card/BACS, net-30/60 for credit), Order Reference, PO Number
         - Bill To: Customer company name, contact name, address
         - Deliver To: Delivery address with site contact
         - Line items table: # | Product | SKU | Qty | Unit Price | Torke Batch ID | Line Total
         - The "Torke Batch ID" column is the Torke differentiator per user decision -- no other fixings supplier includes batch traceability on invoices
         - If line spans multiple batches, show comma-separated batch IDs
         - Subtotal, VAT (20%), Total
         - Payment details section:
           - Card: "Paid by card - {last4 if available}"
           - Credit: "Payment terms: Net {30|60} days. Due: {dueDate}"
           - BACS: "Paid by bank transfer"
         - Footer: Torke registration details placeholder, "Thank you for your business"
      5. Upload PDF to R2 with key `invoices/{invoiceId}.pdf`
      6. Update invoice.pdfKey with R2 key

    **generateProforma(orderId: string): Promise<Buffer>** -- BACS proforma invoice
    - Per user decision: "BACS via proforma invoice with bank details"
    - Same layout as generateInvoice but with key differences:
      - Title: "PROFORMA INVOICE" (not "INVOICE")
      - No invoice number (proforma is not a tax document)
      - Payment section prominently shows bank details:

        - Sort Code: [XX-XX-XX]
        - Account Number: [XXXXXXXX]
        - Payment Reference: {orderNumber}
        - Amount Due: {total formatted in GBP}

      - Placeholder bank details for now (env vars TORKE_BANK_NAME, TORKE_SORT_CODE, TORKE_ACCOUNT_NUMBER, or hardcode placeholders with TODO comment)
    - Upload PDF to R2 with key `proformas/{orderId}.pdf`
    - Store R2 key on order (add proformaKey or reuse a generic field)
    - This proforma is generated when a BACS order is created (called from the orders router or payment service)

    - Use same pdf-lib drawing patterns as cert pack cover page
    - StandardFonts Helvetica and Helvetica-Bold
    - Currency formatting: GBP with 2 decimal places (convert pence to pounds)
    - Share PDF layout helpers between generateInvoice and generateProforma to avoid duplication (header, line items table, footer)
