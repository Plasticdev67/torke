<task type="auto">
  <name>Task 2: Calculations tRPC router (save, load, export)</name>
  <files>
    src/server/trpc/routers/calculations.ts,

  </files>
  <action>
    Create src/server/trpc/routers/calculations.ts with:

    1. generateCalcReference procedure (helper, not exposed):
       - Format: CALC-YYYY-NNNNNN (e.g. CALC-2026-000042)
       - Use MAX-query-in-transaction pattern from order number generation
       - Query max calcReference from calculations table, parse sequence, increment

    2. saveCalculation (protectedProcedure):
       - Input: designInputsSchema (from calc engine validation) + optional projectName, engineerName
       - Re-run calculateAnchorDesign server-side with the inputs (DESIGN-06 validation)
       - Generate calcReference
       - Insert into calculations table: calcReference, userId from ctx.session, inputs (JSON), results (JSON), projectName, engineerName
       - Return: { calcReference, id }

    3. loadCalculation (protectedProcedure):
       - Input: calcReference (string) or id (uuid)
       - Query calculations table where userId = ctx.session.user.id
       - Return: full calculation record (inputs, results, calcReference, projectName, etc.)

    4. listCalculations (protectedProcedure):
       - No input (lists current user's calcs)
       - Query calculations table where userId = ctx.session.user.id, order by createdAt desc, limit 50
       - Return: array of { id, calcReference, projectName, engineerName, overallPass (from results JSON), createdAt }

    5. exportPdf (protectedProcedure):
       - Input: designInputsSchema + optional projectName, engineerName
       - Re-run calculateAnchorDesign server-side (DESIGN-06)
       - Generate or use existing calcReference
       - Call generateCalcReport(inputs, serverResults, ctx.session.user, calcReference)
       - Upload PDF to R2: key = `calc-reports/${calcReference}.pdf`
       - Save calculation to DB (if not already saved)
       - Return: { downloadUrl (presigned R2 URL, 24h expiry), calcReference }

    6. deleteCalculation (protectedProcedure):
       - Input: id (uuid)
       - Delete from calculations where id AND userId match
       - Return: { success: true }

    Add calculationsRouter to the main router in src/server/trpc/routers/index.ts.

    All mutations use protectedProcedure (requires auth) -- this enforces DESIGN-18 (save/export require account).
  </action>

    <automated>npx tsc --noEmit 2>&1 | head -20</automated>
  </verify>
