  <behavior>
    - steel-tension.ts: NRk,s = c * As * fuk (Cl. 7.2.1.3). c=0.85 for chemical, 1.0 for mechanical. As from bolt diameter. fuk from steel grade. gamma_Ms = 1.2 (ductile steel). Returns FailureModeResult with clauseRef "EN 1992-4 Cl. 7.2.1.3"
    - steel-shear.ts: VRk,s (Cl. 7.2.2.3). Without lever arm: VRk,s = 0.5 * As * fuk. gamma_Ms = 1.2. clauseRef "EN 1992-4 Cl. 7.2.2.3"
    - concrete-cone.ts: NRk,c = NRk,c^0 * (Ac,N/Ac,N^0) * psi_s,N * psi_re,N * psi_ec,N (Cl. 7.2.1.4). k1=7.7 cracked/11.0 uncracked (post-installed). gamma_Mc=1.5. Uses projected-areas.ts
    - pull-out.ts: NRk,p (Cl. 7.2.1.5). Generic values: NRk,p = pi * d * hef * tau_Rk where tau_Rk is generic bond strength (chemical: 15 MPa, mechanical: 0 -- uses min of cone/pullout). gamma_Mc=1.5
    - concrete-pryout.ts: VRk,cp = k * NRk,c (Cl. 7.2.2.4). k=1 for hef<60mm, k=2 for hef>=60mm. Uses cone breakout result. gamma_Mc=1.5
    - concrete-edge.ts: VRk,c with psi factors (Cl. 7.2.2.5). Base: VRk,c^0 = k1 * d^alpha * lfixed^beta * sqrt(fck) * c1^1.5. Multiple psi factors for spacing, edge, angle, thickness. gamma_Mc=1.5
    - splitting.ts: NRk,sp (Cl. 7.2.1.6). Check: hmin >= hmin,sp. NRk,sp = NRk,c * psi_h,sp where psi_h,sp = (h/hmin)^(2/3) <= (2*hef/hmin)^(2/3). Uses generic min member thickness
    - combined.ts: Steel interaction: (NEd/NRd,s)^2 + (VEd/VRd,s)^2 <= 1.0. Concrete interaction: (NEd/NRd,c)^1.5 + (VEd/VRd,c)^1.5 <= 1.0 (Table 7.3). Uses min concrete resistance from cone/pullout/splitting for tension, min pryout/edge for shear
    - index.ts: calculateAnchorDesign(inputs) validates via Zod, runs all 8 modes, determines governing mode (highest utilisation), sets overallPass, returns DesignResults with scopeLimitations
    - Test: M12 single anchor centre of C30/37 slab, 15kN tension, 8kN shear -> all modes produce numeric utilisation 0-1. Governing mode identified. Combined interaction evaluated
    - Test: 4-bolt M16 group near edge -> concrete edge breakout utilisation increases due to edge proximity
    - Test: deterministic -- run same inputs 100 times, all results identical
  </behavior>
