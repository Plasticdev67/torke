<objective>
Build the 3D WebGL visualisation using React Three Fiber with parametric geometry driven by the design store. The 3D scene shows an interactive, schematic representation of the anchor connection that updates in real time.

Purpose: The interactive 3D model is a key differentiator vs PROFIS. Engineers can visualise the anchor group, see embedment depth through semi-transparent concrete, view dimension annotations, and toggle failure cone overlays colour-coded by utilisation.

Output: Complete R3F scene integrated into the design page right panel.
</objective>

  <action>
    1. Install dependencies: `npm install @react-three/fiber @react-three/drei three @types/three`

    2. Create AnchorScene.tsx:
       - "use client" directive (WebGL is client-only)
       - R3F Canvas with camera position [0.3, 0.3, 0.3], fov 50
       - Ambient light (0.4 intensity) + directional light (position [5,5,5], 0.8 intensity)
       - drei OrbitControls with damping enabled (DESIGN-10)
       - Render: ConcreteBlock, BasePlate, AnchorGroup, DimensionAnnotations
       - Background colour matching dark theme (#1A1A1A or transparent)
       - Wrap in Suspense with loading fallback (skeleton)

    3. Create BasePlate.tsx:
       - Read plateWidth, plateDepth, plateThickness from useDesignStore (granular selectors)
       - Convert mm to scene units (divide by 1000 for metres)
       - boxGeometry with dimensions from store
       - Position plate at y=0 (top of concrete)
       - meshStandardMaterial: dark grey (#555), metalness 0.7, roughness 0.3
       - Bolt holes as transparent circles (use a simple approach -- darker circles on top face or actual cylinder subtract if feasible)

    4. Create AnchorBolt.tsx:
       - Props: position [x, y, z] in scene units
       - Read anchorDiameter, embedmentDepth from store
       - cylinderGeometry: radius = diameter/2/1000, height = embedment/1000
       - Position: bolt hangs below plate (y = -embedment/2/1000)
       - meshStandardMaterial: metallic silver (#888), metalness 0.8, roughness 0.3
       - Add a washer disc at the top (slightly larger radius, thin height)
       - Add threaded portion above plate (small cylinder extending up)

    5. Create ConcreteBlock.tsx:
       - Read embedmentDepth from store to size the block proportionally
       - Block size: 2x edge distance + plate width on each axis, depth = 1.5x embedment
       - boxGeometry with calculated dimensions
       - meshStandardMaterial: light grey (#AAAAAA), transparent=true, opacity=0.25, side=DoubleSide
       - Position below plate at appropriate y offset
       - Add subtle wireframe overlay for edge visibility

    6. Create AnchorGroup.tsx:
       - Read groupPattern, spacing from store
       - Import getAnchorPositions from calc engine geometry (or inline simple position logic)
       - Map positions to AnchorBolt components with correct [x, 0, z] positions
       - Handles: single (1 bolt centre), 2x1 (2 bolts), 2x2 (4 bolts), 3x2 (6 bolts)

    7. Create DimensionAnnotation.tsx:
       - Use drei Html component for text labels positioned in 3D space
       - Use drei Line for dimension lines (thin lines with arrows/ticks at ends)
       - Show: embedment depth (vertical), edge distances c1/c2 (horizontal from bolt to concrete edge), spacing s1/s2 (between bolts when group)
       - Text style: white text, small font, background semi-transparent dark pill
       - Dimension values update from store (live)
       - Add a toggleable prop (visible by default, can be hidden if view is cluttered)

    All components use React.memo and granular Zustand selectors (select individual fields, not the whole inputs object) to prevent unnecessary R3F re-renders per research pitfall guidance.
  </action>
