<objective>
Build the customer cert portal as a new "Certifications" tab within the existing account dashboard. Contractors can search their certs by order number, Torke batch ID, product code, or date range, download individual 3.1 cert PDFs, download per-order cert packs, and select multiple orders for a bulk ZIP download.

Purpose: Enables contractors to self-serve cert retrieval for audits, project handovers, and site inspections -- core traceability differentiator.
Output: Certifications tab with search, results, and download functionality. New certs tRPC router, ZIP service, bulk download API route.
</objective>

<task type="auto">
  <name>Task 1: Cert search router, ZIP service, and bulk download API</name>
  <files>src/server/db/schema/share-tokens.ts, src/server/db/schema/index.ts, src/server/trpc/routers/certs.ts, src/server/trpc/router.ts, src/server/services/zip-service.ts, src/app/api/certpack/bulk/route.ts</files>
  <action>
1. Install jszip: `npm install jszip`

2. Create `src/server/db/schema/share-tokens.ts` with `orderShareTokens` table (needed by Plan 02 but creating now to avoid schema migration conflicts):
   - id: uuid PK
   - token: uuid unique not null (defaultRandom)
   - orderId: uuid FK to orders.id not null
   - userId: uuid not null (creator)
   - projectName: varchar(500) optional
   - createdAt: timestamp defaultNow
   - lastAccessedAt: timestamp optional
   - Add relations: orderShareTokens -> orders

3. Export from `src/server/db/schema/index.ts`.

4. Create `src/server/trpc/routers/certs.ts`:
   - `search` protectedProcedure: accepts orderNumber?, batchId?, productCode?, dateFrom?, dateTo?, limit (default 20), offset (default 0). Queries orders for current user joining through orderLines -> orderLineAllocations -> batches -> products. Apply ILIKE filters conditionally. Return array of {orderId, orderNumber, orderDate, status, certPackKey, lines: [{productName, productSku, torkeBatchId, quantity, batchCertKey}]}. Group results by order.
   - `orderCertDetail` protectedProcedure: accepts orderId. Returns full allocation + batch detail for one order (reuses myOrderDetail pattern but focused on cert data).

5. Register certsRouter in main router.

6. Create `src/server/services/zip-service.ts`:
   - `generateBulkCertZip(certKeys: Array<{key: string; filename: string}>): Promise<Buffer>` using JSZip.
   - Fetches each PDF from R2 via downloadFile, adds to zip, returns buffer.
   - Max 20 items enforced.

7. Create `src/app/api/certpack/bulk/route.ts`:
   - POST endpoint accepting `{orderIds: string[]}` body.
   - Validates user owns all orders. Fetches certPackKeys. Calls generateBulkCertZip. Returns ZIP as application/zip response with Content-Disposition attachment header.
   - Limit to 20 orders max, return 400 if exceeded.
  </action>
  <verify>
    <automated>npx vitest run --reporter=verbose 2>&1 | tail -5</automated>
  </verify>
  <done>certs.search returns filtered results for authenticated user. Bulk ZIP endpoint generates valid ZIP from R2 cert packs. Share tokens table exists in schema.</done>
</task>
