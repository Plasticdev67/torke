<task type="auto">
  <name>Task 2: Define complete Phase 1 database schema in Drizzle ORM</name>
  <files>
    src/server/db/index.ts,
    src/server/db/schema/products.ts,
    src/server/db/schema/batches.ts,
    src/server/db/schema/users.ts,

    src/server/db/schema/index.ts,
    src/server/db/migrate.ts,
    drizzle.config.ts
  </files>
  <action>
    1. Create `src/server/db/index.ts` — Drizzle client using `postgres` driver:
    ```typescript
    import { drizzle } from 'drizzle-orm/postgres-js';
    import postgres from 'postgres';
    import * as schema from './schema';
    const client = postgres(process.env.DATABASE_URL!);
    export const db = drizzle(client, { schema });
    ```

    2. Create schema files exactly as specified in the research (Section 2):

    **products.ts:** `categories` table (id, name, slug, parentId, description, sortOrder, createdAt) and `products` table (id, name, slug, sku, categoryId, description, technicalSpecs JSONB, diameter, material, length_mm, finish, loadClass, etaReference, datasheetUrl, images JSONB, isActive, createdAt, updatedAt). Include `categoryEnum` for top-level types. Add Drizzle relations.

    **batches.ts:** `batchStatusEnum` (pending, available, quarantined, depleted). Tables: `suppliers` (id, name, code, contactInfo JSONB, createdAt), `millCerts` (id, heatNumber, millName, documentUrl, chemicalComposition JSONB, mechanicalProperties JSONB, createdAt), `supplierBatches` (id, supplierId FK, supplierBatchNumber, millCertId FK, manufacturerCertUrl, productId FK, quantityReceived, productionDate, createdAt), `batches` (id, torkeBatchId unique format TRK-YYYYMMDD-NNNN, supplierBatchId FK, productId FK, goodsInDate, receivedBy FK to users, inspectionNotes, quantity, quantityAvailable, quantityReserved, status enum, expiryDate, createdAt, updatedAt). Add all Drizzle relations connecting the chain.

    **stock.ts:** `stockItems` table (id, batchId FK, productId FK, quantity, binLocation, createdAt, updatedAt).

    **verification.ts:** `verificationTokens` table (id, token UUID unique defaultRandom, batchId FK, createdAt, lastAccessedAt). The `token` field is the opaque UUID used in QR URLs — NEVER the batch ID.

    **allocations.ts:** `orderLineAllocations` table (id, orderLineId UUID — FK added in Phase 2, batchId FK, quantity, allocatedAt, pickedAt, dispatchedAt). This is the many-to-many join table for TRACE-08.

    **users.ts:** `userProfiles` table (id, userId text unique — FK to Better Auth user, companyName, phone, role default 'customer', createdAt).

    **index.ts:** Re-export all schema tables and relations from a single entry point.

    3. Create `drizzle.config.ts` pointing schema to `./src/server/db/schema/*`, output to `./drizzle`, dialect `postgresql`.

    4. Create `src/server/db/migrate.ts` — a script that runs `drizzle-kit migrate` programmatically.

    5. Generate initial migration: `npx drizzle-kit generate`

    IMPORTANT: All tables use `uuid('id').primaryKey().defaultRandom()`. All timestamps use `timestamp('...').defaultNow().notNull()`. The `torkeBatchId` format is `TRK-YYYYMMDD-NNNN` and is the INTERNAL identifier — never used in QR URLs.
  </action>
  <verify>npx drizzle-kit generate succeeds and creates migration SQL files. tsc --noEmit passes with all schema types correct. If a local database is available, run npx drizzle-kit migrate to verify tables are created.</verify>
  <done>All Phase 1 database tables defined in Drizzle ORM with correct types, relations, enums, and foreign keys. Migration files generated. Schema supports the full traceability chain: products -> batches -> supplierBatches -> millCerts, with verification tokens and order line allocations.</done>
</task>
