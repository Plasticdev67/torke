  <action>
    **DESIGN DIRECTION — THIS MUST BE PREMIUM:**
    The site must feel like a top-tier construction tech brand. Think: Tesla's website meets Hilti's product catalogue. Dark, confident, angular. Not a generic SaaS template. Specific design language:
    - **Background:** Near-black (#0A0A0A) or very dark grey (#111111) base. NOT pure white.
    - **Cards/surfaces:** Dark grey (#1A1A1A to #2D2D2D) with subtle borders (#333).
    - **Text:** White (#FFFFFF) for headings, light grey (#B3B3B3) for body text.
    - **Accent:** Torke red (#C41E3A) used sparingly — CTAs, active states, hover accents, category indicators. NOT as background colour.
    - **Typography:** Clean sans-serif (Inter or Geist). Large, confident headings. Technical data in monospace or tabular figures.
    - **Spacing:** Generous — let the products breathe. White space conveys premium.
    - **Imagery:** Product images on dark backgrounds pop. Use subtle shadows/glows.
    - **Micro-interactions:** Smooth hover transitions on cards (scale, shadow lift). Search results fade in. Filter counts animate.

    1. **Global CSS** (`src/app/globals.css`):
    Define CSS custom properties for the Torke dark theme. Override shadcn/ui defaults. Set body background to near-black. Define smooth transitions for interactive elements.

    2. **Header** (`src/components/layout/Header.tsx`):
    - Torke logo (angular "T" monogram) on left — links to home
    - Main navigation: Products (dropdown with three categories), About, Contact
    - Search bar integrated into header — prominent, with magnifying glass icon
    - Account button (login/register if unauthenticated, account dropdown if authenticated)
    - Sticky header with blur backdrop on scroll
    - Mobile: hamburger menu triggers Sheet (shadcn/ui) with full navigation
    - The header should feel like Hilti's but darker and more refined

    3. **Category Navigation** (`src/components/products/CategoryNav.tsx`):
    - Horizontal category bar or sidebar showing three top-level categories
    - Each category shows product count
    - Active category highlighted with Torke red accent
    - Sub-categories displayed on hover or click

    4. **Footer** (`src/components/layout/Footer.tsx`):
    - Dark grey background slightly lighter than body
    - Torke logo, company info, links to key pages
    - Contact details (email, phone)
    - Copyright notice
    - Clean, minimal — not cluttered

    5. **Shop layout** (`src/app/(shop)/layout.tsx`):
    - Wraps Header + children + Footer
    - Sets page-level metadata (title template: "{page} | Torke")

    6. **tRPC products router** (`src/server/trpc/routers/products.ts`):
    - `products.list` — query products with optional category filter, pagination (offset/limit), sorting. Returns products with category info. Uses direct Drizzle queries (Server Components can call this directly).
    - `products.getBySlug` — query single product by slug with full specs, category, images. For product detail page.
    - `products.listByCategory` — query products filtered by categorySlug. Return with facet counts from DB (count per diameter, material, etc.) as a fallback when Meilisearch is unavailable.

    7. **tRPC search router** (`src/server/trpc/routers/search.ts`):
    - `search.products` — takes `query` (string), `filters` (optional object with categorySlug, diameter, material, finish, loadClass), `limit`, `offset`. Calls Meilisearch with faceted search. Returns `{ hits, facetDistribution, totalHits, processingTimeMs }`.
    - Must return results in under 200ms (SHOP-05).

    8. **Wire routers into appRouter** in `src/server/trpc/router.ts`.

    9. **useDebounce hook** (`src/lib/hooks/useDebounce.ts`):
    Standard debounce hook with configurable delay (default 200ms).
