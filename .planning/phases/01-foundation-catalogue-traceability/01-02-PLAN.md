Output: Structured product data in JSON, database seed scripts, Meilisearch indexing scripts, downloaded product images and datasheets.

<task type="auto">
  <name>Task 1: Build Playwright scraper for proventure.co.uk product catalogue</name>
  <files>scripts/scrape-proventure.ts, scripts/tsconfig.json, data/scraped-products.json</files>
  <action>
    1. Create `scripts/tsconfig.json` for running TypeScript scripts with ts-node/tsx:
    ```json
    { "extends": "../tsconfig.json", "compilerOptions": { "module": "commonjs", "outDir": "../dist/scripts" } }
    ```

    2. Install scraping dependencies: `npm install -D playwright tsx`; run `npx playwright install chromium`.

    3. Create `scripts/scrape-proventure.ts`:

    **Step 1: Discover category structure.**
    Navigate to proventure.co.uk. Find the main navigation/category pages. Identify the three top-level categories: chemical anchors, mechanical anchors, general fixings. Collect all category URLs and sub-category URLs.

    **Step 2: Scrape product listing pages.**
    For each category page, collect all product links. Handle pagination if present (check for "next page" buttons or infinite scroll).

    **Step 3: Scrape each product detail page.**
    For each product URL, extract:
    - Product name (from h1 or title element)
    - SKU/product code (from product info section)
    - Description (from product description section)
    - Technical specifications (from spec table — key/value pairs)
    - Image URLs (all product images — main image and gallery)
    - Datasheet/PDF download URL (if present)
    - Category path (breadcrumb or category label)
    - Price (if displayed — may not be public on B2B site)
    - ETA reference (if present in specs)

    Use `page.waitForSelector()` to handle JS-rendered content. Add delays between requests (1-2 seconds) to be polite to the server. Use `try/catch` per product to continue on errors.

    **Step 4: Save raw data.**
    Write all scraped products to `data/scraped-products.json` with the `ScrapedProduct` interface:
    ```typescript
    interface ScrapedProduct {
      sourceUrl: string;
      name: string;
      sku: string;
      category: string;
      subcategory?: string;
      description: string;
      technicalSpecs: Record<string, string>;
      imageUrls: string[];
      datasheetUrl?: string;
      etaReference?: string;
      scrapedAt: string;
    }
    ```

    Add console logging for progress: "Scraping product 1/N: ProductName".

    IMPORTANT: The scraper is a one-time tool, not production code. Prioritise data completeness over code elegance. If Proventure's site structure is unexpected, adapt the selectors. The goal is to get ALL products with ALL available data.
  </action>
  <verify>Run `npx tsx scripts/scrape-proventure.ts` and confirm data/scraped-products.json is created with product entries. Spot-check 5 products for completeness (name, sku, description, specs, images).</verify>
  <done>Raw product data scraped from proventure.co.uk and saved as structured JSON. All categories covered. Each product has name, SKU, description, specs, images, and datasheet URL where available.</done>
</task>

<task type="auto">
  <name>Task 2: Transform scraped data into Torke schema and download assets</name>
  <files>scripts/transform-products.ts, scripts/download-assets.ts, data/transformed-products.json, data/categories.json</files>
  <action>
    1. Create `scripts/transform-products.ts` that reads `data/scraped-products.json` and outputs `data/transformed-products.json`.

    **Transformation rules:**
    - Map Proventure categories to Torke's three top-level categories plus sub-categories. Create `data/categories.json` with the category tree.
    - Generate URL-safe slugs from product names using `slugify()` (e.g. "M12 x 120mm Chemical Anchor Stud A4 Stainless" -> "m12-x-120mm-chemical-anchor-stud-a4-stainless"). Ensure uniqueness — append SKU if collision.
    - Generate Torke SKUs: `TRK-{TYPE_CODE}-{DIAMETER}-{LENGTH}-{MATERIAL_CODE}`. Type codes: CA (chemical anchor), MA (mechanical anchor), GF (general fixing). Material codes: A2, A4, ZP (zinc plated), SS (stainless steel), CS (carbon steel). Example: `TRK-CA-M12-120-A4`.
    - Extract facetable fields from specs/description:
      - `diameter`: Extract M8, M10, M12, M16, M20, M24 patterns
      - `material`: Extract A2, A4, stainless, zinc plated, galvanised patterns
      - `length`: Extract length in mm (integer)
      - `finish`: Extract finish descriptions (zinc plated, hot-dip galvanised, A2 stainless, A4 stainless)
      - `loadClass`: Extract load class if present in specs
    - Keep raw `technicalSpecs` as JSONB for type-specific data
    - Validate: reject products missing name or SKU, log warnings for missing images or specs

    2. Create `scripts/download-assets.ts`:
    - Download all product images from Proventure, save to `data/assets/images/{slug}/` directory
    - Download all datasheet PDFs, save to `data/assets/datasheets/{slug}.pdf`
    - For each downloaded asset, record the local file path in the transformed product data (these will be uploaded to R2 later, or served locally in dev)
    - Add retry logic (3 attempts) and continue on failure
    - Log progress: "Downloaded image 1/N for product: ProductName"

    Output format for `data/transformed-products.json`:
    ```typescript
    interface TransformedProduct {
      name: string;
      slug: string;
      sku: string;
      categorySlug: string;
      description: string;
      technicalSpecs: Record<string, string>;
      diameter: string | null;
      material: string | null;
      lengthMm: number | null;
      finish: string | null;
      loadClass: string | null;
      etaReference: string | null;
      localImages: string[];
      localDatasheetPath: string | null;
      isActive: true;
    }
    ```
  </action>
  <verify>Run transform script. Check data/transformed-products.json: every product has a valid slug, Torke SKU, categorySlug from the three categories, and extracted facet fields. Run download-assets script and verify images/PDFs are saved locally.</verify>
  <done>Product data transformed to Torke schema with generated slugs, SKUs, and extracted facet fields. Assets downloaded locally. Data is ready for database seeding.</done>
</task>

<task type="auto">
  <name>Task 3: Create database seed and Meilisearch indexing scripts</name>
  <files>scripts/seed-products.ts, scripts/seed-search-index.ts</files>
  <action>
    1. Create `scripts/seed-products.ts`:
    - Import the Drizzle `db` client and schema from `src/server/db`
    - Read `data/categories.json` and `data/transformed-products.json`
    - Insert categories first (top-level, then sub-categories with parentId references)
    - Insert products with categoryId FK lookups
    - For images and datasheets: if R2 is configured, upload to R2 and store the R2 key. If not (local dev), store the local file path as a placeholder.
    - Use `db.insert().values().onConflictDoNothing()` to make the script idempotent (can re-run safely)
    - Log progress: "Seeded N categories, M products"
    - At the end, log a summary: total products per category, products with missing images, products with datasheets

    2. Create `scripts/seed-search-index.ts`:
    - Import the Meilisearch client from `src/server/search`
    - Configure the 'products' index with:
      - Searchable attributes: name, sku, description, etaReference, categoryName
      - Filterable attributes: categorySlug, diameter, material, finish, loadClass, isActive
      - Sortable attributes: name, sku
    - Query all products from PostgreSQL (joined with category names)
    - Bulk-index all products into Meilisearch: `await meili.index('products').addDocuments(products)`
    - Verify indexing: query the index stats and log document count

    3. Add npm scripts to `package.json`:
    ```json
    "scripts": {
      "scrape": "tsx scripts/scrape-proventure.ts",
      "transform": "tsx scripts/transform-products.ts",
      "download-assets": "tsx scripts/download-assets.ts",
      "seed:db": "tsx scripts/seed-products.ts",
      "seed:search": "tsx scripts/seed-search-index.ts",
      "seed": "npm run seed:db && npm run seed:search"
    }
    ```
  </action>
  <verify>
    1. With database running: `npm run seed:db` inserts all products. Query `SELECT count(*) FROM products` confirms expected count.
    2. With Meilisearch running: `npm run seed:search` indexes all products. Query Meilisearch: `curl http://localhost:7700/indexes/products/stats` shows matching document count.
    3. Test search: `curl 'http://localhost:7700/indexes/products/search?q=M12&facets=["diameter","material"]'` returns results with facet counts.
  </verify>
  <done>Database seeded with full product catalogue. Meilisearch index configured and populated. Faceted search returns correct results. Seed scripts are idempotent and can be re-run safely.</done>
